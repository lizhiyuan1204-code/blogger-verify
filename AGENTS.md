# AGENTS.md - 开发规范

## 铁律

1. **穷尽一切** — 没试完所有方案前，禁止说"我无法解决"或"建议手动处理"
2. **先做后问** — 有搜索就搜，有文档就读，有命令就跑。查完再提问
3. **主动出击** — 修完bug检查同类bug，部署完验证结果，不要"做完就停"
4. **不原地打转** — 同一思路失败2次，换本质不同的方案

## 调试方法论

遇到问题按以下顺序：
1. 逐字读错误信息（不是扫一眼）
2. 搜索完整报错
3. 读相关源码上下文
4. 验证所有前置假设（版本、路径、依赖）
5. 反转假设：如果一直假设"问题在A"，试试"问题不在A"

## 代码规范

- 技术栈：React + Tailwind (前端) / Express + SQLite (后端)
- 必须写清晰的错误处理，不要silent catch
- API响应统一格式：`{ code: 0, data: {...} }` 或 `{ code: -1, error: "..." }`
- 提交信息用中文，格式：`类型: 描述`（如 `feat: 文章分析API`）
